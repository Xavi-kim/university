# ✅ ИСПРАВЛЕНО: Каталог теперь доступен без авторизации

## 🐛 Проблема

Вкладка "Каталог" (`/catalog`) перенаправляла на страницу аутентификации вместо показа доступных курсов.

**Симптом:** При попытке открыть `/catalog` без входа в систему, происходил редирект на `/auth/login`.

---

## 🔍 Причина

В `SecurityConfig.java` URL `/catalog` не был добавлен в список публичных страниц `.permitAll()`.

**Было:**
```java
.requestMatchers(
    "/",
    "/auth/register",
    "/auth/login",
    "/register",
    "/login",
    "/css/**",
    "/js/**",
    "/images/**",
    "/api/**"
).permitAll()
// ❌ /catalog отсутствует
```

Из-за этого `/catalog` попадал под правило:
```java
.anyRequest().authenticated()  // Требует авторизацию
```

---

## ✅ Решение

Добавлены все публичные страницы в список `permitAll()`:

```java
.requestMatchers(
    "/",
    "/home",
    "/catalog",        // ✅ Каталог курсов
    "/about",          // ✅ О системе
    "/browse",         // ✅ API Browser (редирект на /catalog)
    "/api-docs",       // ✅ API документация
    "/auth/register",
    "/auth/login",
    "/register",
    "/login",
    "/css/**",
    "/js/**",
    "/images/**",
    "/api/**"
).permitAll()
```

Также добавлена явная проверка для студенческих страниц:
```java
.requestMatchers("/student/**").hasAnyRole("STUDENT", "ADMIN")
```

### Публичные страницы (без авторизации):
✅ `/` - Главная страница
✅ `/catalog` - Каталог курсов, преподавателей, университетов
✅ `/about` - О системе
✅ `/api-docs` - Документация REST API
✅ `/browse` - Редирект на `/catalog`
✅ `/auth/login` - Вход
✅ `/auth/register` - Регистрация

### Страницы требующие авторизации:
🔒 `/admin/**` - Только для администраторов (ROLE_ADMIN)
🔒 `/student/**` - Для студентов и администраторов (ROLE_STUDENT, ROLE_ADMIN)
🔒 `/dashboard` - Требует авторизацию
