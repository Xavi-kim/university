# 🔧 РЕШЕНИЕ ПРОБЛЕМЫ: Порт 8080 занят

## ❌ Проблема

```
Web server failed to start. Port 8080 was already in use.
```

Это означает, что на порту 8080 уже работает другой процесс.

---

## ✅ АВТОМАТИЧЕСКОЕ РЕШЕНИЕ

### Вариант 1: Используйте start.bat

```bat
start.bat
```

Этот файл **автоматически**:
- Находит все процессы на порту 8080
- Останавливает их
- Запускает новый экземпляр приложения
- Открывает браузер

---

## ✅ РУЧНОЕ РЕШЕНИЕ

### Шаг 1: Найти процесс на порту 8080

**PowerShell:**
```powershell
netstat -ano | Select-String ":8080"
```

**Вывод будет выглядеть так:**
```
TCP    0.0.0.0:8080           0.0.0.0:0              LISTENING       12360
```

Запомните **номер процесса** (в примере: `12360`)

---

### Шаг 2: Остановить процесс

**PowerShell:**
```powershell
Stop-Process -Id 12360 -Force
```

Замените `12360` на ваш номер процесса.

---

### Шаг 3: Проверить, что порт свободен

```powershell
netstat -ano | Select-String ":8080"
```

Если команда ничего не возвращает - порт свободен! ✅

---

### Шаг 4: Запустить приложение

**Из командной строки:**
```bat
start.bat
```

**Из IDE (IntelliJ IDEA):**
1. Нажмите зелёную стрелку ▶️
2. Или нажмите Shift + F10

---

## 🔍 ПОЧЕМУ ЭТО ПРОИСХОДИТ?

### Причина 1: Приложение уже запущено

Вы случайно запустили приложение дважды:
- Один раз через IDE
- Один раз через командную строку

**Решение:** Используйте только один способ запуска.

---

### Причина 2: Предыдущий процесс не остановился

При остановке через IDE, процесс Java не завершился полностью.

**Решение:** Остановите вручную через диспетчер задач или PowerShell.

---

### Причина 3: Tomcat работает отдельно

У вас установлен Apache Tomcat, который уже использует порт 8080.

**Решение 1 - Изменить порт приложения:**

Отредактируйте `src/main/resources/application.yml`:

```yaml
server:
  port: 8081  # Изменено с 8080 на 8081
```

Теперь приложение будет доступно на: http://localhost:8081/

**Решение 2 - Остановить Tomcat:**

```bat
C:\tomcat10\bin\shutdown.bat
```

## 🛠️ ДОПОЛНИТЕЛЬНЫЕ КОМАНДЫ

### Найти все Java-процессы
```powershell
Get-Process -Name java
```

### Остановить все Java-процессы
```powershell
Get-Process -Name java | Stop-Process -Force
```

### Проверить, какие порты используются
```powershell
netstat -ano | Select-String "LISTENING"
```

## 🚨 ЕСЛИ НИЧЕГО НЕ ПОМОГЛО

### Вариант 1: Изменить порт

**application.yml:**
```yaml
server:
  port: 9090
```

Теперь открывайте: http://localhost:9090/

---

### Вариант 2: Найти, что занимает порт

```powershell
netstat -ano | Select-String ":8080"
Get-Process -Id ПИД_ПРОЦЕССА
```

Это покажет, какая программа занимает порт.

---

### Вариант 3: Перезагрузить компьютер

Самый простой способ освободить все порты. 😊
